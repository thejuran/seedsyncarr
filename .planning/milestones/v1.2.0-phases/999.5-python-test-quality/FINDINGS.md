### BUG-11/12: Logger handler leaked every test — no removeHandler in tearDown (score: 85-88)
- **Files:** `test_auto_queue.py:281-287`, `test_model_builder.py:16-24`, `test_remote_scanner.py:28-33`, `test_lftp.py:114-119`, `test_multiprocessing_logger.py`
- **Issue:** Each `setUp` adds a `StreamHandler` to root or named logger without removal. After N tests, every log line prints N times.
- **Fix:**
  ```diff
    def setUp(self):
  +     self._handler = logging.StreamHandler(sys.stdout)
  -     handler = logging.StreamHandler(sys.stdout)
  -     logger.addHandler(handler)
  +     logger.addHandler(self._handler)
  
  + def tearDown(self):
  +     logger.removeHandler(self._handler)
  +     self._handler.close()
  ```
- **Cross-ref:** Overlaps with TEST-HARDENING-REVIEW.md W-05 (logger fixture) — this is the broader class-level version.
