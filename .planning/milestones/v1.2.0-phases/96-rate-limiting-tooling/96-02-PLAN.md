    **TOOL-01 (D-07): Fix js-nosql-injection-where (lines 267-280)**

    Convert the single `pattern:` key to a `patterns:` block and add a `metavariable-regex` constraint. The final rule block must be:

    ```yaml
      - id: js-nosql-injection-where
        patterns:
          - pattern: |
              $COLLECTION.$WHERE($INPUT)
          - metavariable-regex:
              metavariable: $WHERE
              regex: '^\$where$'
        message: >
          The $where operator evaluates JavaScript on the server. Never pass
          user-controlled data to $where. Use standard query operators instead.
        severity: ERROR
        languages: [javascript, typescript]
        metadata:
          cwe: "CWE-943"
          owasp: "A03:2021 Injection"
          confidence: HIGH
          impact: CRITICAL
          category: security
    ```

    Key details:
    - Replace `pattern:` (singular) with `patterns:` (plural) at the rule level
    - The original `pattern:` block becomes a child `- pattern:` under `patterns:`
    - Add `- metavariable-regex:` as a sibling to `- pattern:` under `patterns:`
    - Use single-quoted YAML string `'^\$where$'` for the regex to prevent YAML from interpreting `$where` as a variable reference
    - Keep all other fields (message, severity, languages, metadata) unchanged

    **TOOL-02 (D-08): Fix js-xss-eval-user-input (lines 87-110)**

    Add six `pattern-not` entries as siblings to the existing `pattern-not: eval("...")` at line 98-99. Insert them immediately after line 99. The additions are:

    ```yaml
          - pattern-not: |
              setTimeout(($ARGS) => $BODY, ...)
          - pattern-not: |
              setTimeout(() => $BODY, ...)
          - pattern-not: |
              setTimeout(function $NAME($PARAMS) { $BODY }, ...)
          - pattern-not: |
              setInterval(($ARGS) => $BODY, ...)
          - pattern-not: |
              setInterval(() => $BODY, ...)
          - pattern-not: |
              setInterval(function $NAME($PARAMS) { $BODY }, ...)
    ```

    Key details:
    - Each `pattern-not:` is a sibling under the top-level `patterns:` key (same indent level as the existing `- pattern-not: eval("...")`)
    - Both zero-arg `() => $BODY` and parameterized `($ARGS) => $BODY` forms are needed (Semgrep treats them as distinct patterns per Assumption A3)
    - The existing `pattern-either:` and `pattern-not: eval("...")` entries remain unchanged
    - Keep all other fields (message, severity, languages, metadata) unchanged
