## Coverage Gaps

### GAP-01: SSE streaming has no integration test coverage (high priority)
- **Issue:** 3 integration tests permanently skipped (`webtest doesn't support SSE streaming`):
  - `tests/integration/test_web/test_handler/test_stream_model.py`
  - `tests/integration/test_web/test_handler/test_stream_status.py`
  - `tests/integration/test_web/test_handler/test_stream_log.py`
- **Impact:** SSE handlers are the real-time backbone of the Angular frontend. Changes to SSE format or session lifecycle are invisible to the test suite. Angular unit tests (`base-stream.service.spec.ts`, `model-file.service.spec.ts`) test client-side parsing but rely on mocked event sources. No layer tests the full server-to-client SSE contract.
- **Recommendation:** Evaluate `httpx` AsyncClient with SSE support, or a thin integration harness that starts the real WSGI app and connects via EventSource.

### GAP-02: Logs page has no E2E coverage
- **Issue:** `src/e2e/urls.ts` defines `Paths.LOGS` but there is no `logs.page.ts` or `logs.page.spec.ts`. The Logs page is exercised only incidentally (nav spec confirms "Logs" appears in the nav bar). Angular has `logs-page.component.spec.ts` for unit coverage.
- **Recommendation:** Create `logs.page.ts` page object and `logs.page.spec.ts` with at least: page loads, log entries render, auto-scroll behavior.

### GAP-03: Settings form fields have no E2E coverage
- **Issue:** `settings.page.spec.ts` contains a single test (nav active state). `settings-error.spec.ts` adds one Sonarr error scenario. The Settings page form — LFTP connection fields, controller intervals, encryption toggle, Radarr integration — has no E2E coverage. All covered only at Angular unit level.
- **Recommendation:** Add E2E specs for the critical path: set remote host/user, toggle encryption, save and verify persistence after restart.

### GAP-04: Webhook end-to-end absent
- **Issue:** No scenario tests an actual Sonarr/Radarr webhook POST flowing through the web layer into the controller. `settings-error.spec.ts` tests the connection test UI only. Unit-level `test_webhook_handler.py` exists but is isolated from the real web app.
- **Recommendation:** Add integration or E2E test that POSTs a Sonarr/Radarr webhook payload and asserts the controller responds with the expected queue/import action.

### GAP-05: DeleteRemoteProcess only tested via mocks
- **Issue:** `DeleteRemoteProcess` is tested only indirectly via `test_file_operation_manager.py` at the mock level. The class is patched out. No unit tests exercise the actual SSH command construction, error handling, or deletion paths. The extract equivalent (`test_extract_process.py`) exists.
- **Recommendation:** Create `test_delete_process.py` mirroring the extract process test structure.

### GAP-06: ActiveScanner has no dedicated test file
- **Issue:** `ActiveScanner` appears referenced only by `test_scan_manager.py` tests. The three sibling scanner modules (`local_scanner`, `remote_scanner`, `scanner_process`) each have their own spec files. `ActiveScanner` does not.
- **Recommendation:** Create `test_active_scanner.py` with unit tests for the scan scheduling and result aggregation logic.
