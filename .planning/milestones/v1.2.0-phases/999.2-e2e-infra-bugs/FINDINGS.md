## Warning

### BUG-05: SERVER_UP/SCAN_DONE uninitialized before polling loops (score: 95)
- **File:** `src/docker/test/e2e/run_tests.sh:8-24`
- **Issue:** Variables only assigned inside `while` loop body. If curl fails on every iteration, post-loop check references unset variables.
- **Fix:**
  ```diff
  + SERVER_UP='False'
  + SCAN_DONE='False'
    END=$((SECONDS+30))
  ```

### BUG-06: configure service depends on myapp without health condition (score: 90)
- **File:** `src/docker/test/e2e/compose.yml:34-35`
- **Issue:** Uses bare `depends_on` (default: `service_started`); configure may run before myapp HTTP server is ready. The `wait-for-it.sh` mitigates but is a brittle second layer.
- **Fix:**
  ```diff
    configure:
      depends_on:
  -     - myapp
  +     myapp:
  +       condition: service_healthy
  ```

### BUG-08: Bare except: swallows SystemExit/KeyboardInterrupt (score: 90)
- **File:** `src/docker/test/e2e/parse_status.py:21`
- **Issue:** Catches everything including SystemExit. CI cancellation silently prints 'False' and exits 0.
- **Fix:**
  ```diff
  - except:
  + except (json.JSONDecodeError, KeyError, TypeError, ValueError):
        print('False')
  ```

### BUG-07: sleep 2 race after restart (score: 82)
- **File:** `src/docker/test/e2e/configure/setup_seedsyncarr.sh:32-34`
- **Issue:** After posting restart, `sleep 2` then `wait-for-it.sh` may connect to the *old* process that hasn't shut down yet.
- **Fix:**
  ```diff
  - sleep 2
  - ./wait-for-it.sh myapp:8800 -t 60 -s -- echo "Seedsync app is up (after configuring)"
  + ./wait-for-it.sh myapp:8800 -t 10 --invert-result -- true 2>/dev/null || true
  + ./wait-for-it.sh myapp:8800 -t 60 -s -- echo "Seedsync app is up (after configuring)" \
  +   || { echo "ERROR: myapp:8800 not available within 60s"; exit 1; }
  ```
