**3. Replace PasswordAuthentication yes with no using sed (replace lines 28-30):**

Replace the current block:
```dockerfile
RUN echo "PasswordAuthentication yes" >> /etc/ssh/sshd_config && \
    echo "KbdInteractiveAuthentication yes" >> /etc/ssh/sshd_config && \
    echo "UsePAM yes" >> /etc/ssh/sshd_config
```
with:
```dockerfile
# DOCKSEC-03: Disable password authentication — key-only auth
RUN sed -i 's/^#\?PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config && \
    sed -i 's/^#\?KbdInteractiveAuthentication.*/KbdInteractiveAuthentication no/' /etc/ssh/sshd_config && \
    sed -i 's/^#\?UsePAM.*/UsePAM no/' /etc/ssh/sshd_config
# Lock the user password to prevent any password-based login
RUN passwd -l seedsyncarrtest
```

IMPORTANT: Use `sed -i` to replace existing lines, NOT `echo >>` append. The default sshd_config in Debian Bookworm may contain commented-out `#PasswordAuthentication yes` lines. Appending creates conflicting entries (see Research Pitfall 3).

**4. Replace root group with dedicated testgroup (replace line 35):**

Change:
```dockerfile
RUN usermod -a -G root seedsyncarrtest
```
to:
```dockerfile
# DOCKSEC-01: Dedicated group instead of root group
RUN groupadd testgroup && \
    usermod -a -G testgroup seedsyncarrtest && \
    usermod -a -G testgroup root
```

**5. Change StrictHostKeyChecking from no to accept-new (replace line 25):**

Change:
```dockerfile
RUN echo "StrictHostKeyChecking no\nUserKnownHostsFile /dev/null\nLogLevel=quiet" > /root/.ssh/config
```
to:
```dockerfile
# DOCKSEC-06: accept-new rejects changed keys (MITM protection) while accepting new ones
RUN printf "StrictHostKeyChecking accept-new\nUserKnownHostsFile /dev/null\nLogLevel=quiet\n" > /root/.ssh/config
```

Note: Change `echo` to `printf` because `echo` interprets `\n` differently across shells. `printf` is reliable.

**6. Add setcap and sshd ownership transfer (add AFTER the USER root line, BEFORE EXPOSE 22):**

Insert before `EXPOSE 22`:
```dockerfile
# DOCKSEC-04: Allow non-root sshd to bind port 22
RUN setcap cap_net_bind_service=+ep /usr/sbin/sshd

# DOCKSEC-04: Transfer sshd runtime ownership to sshdaemon
RUN chown sshdaemon /var/run/sshd && \
    chown sshdaemon:sshdaemon /etc/ssh/ssh_host_* && \
    chmod 700 /etc/ssh/ssh_host_*
```

CRITICAL: `setcap` MUST be the last step that touches `/usr/sbin/sshd` -- any later `apt-get install` of openssh-server would wipe the capability (Research Pitfall 4).

**7. Edit `src/docker/test/python/entrypoint.sh` (line 7):**

Change:
```bash
/usr/sbin/sshd -D &
```
to:
```bash
gosu sshdaemon /usr/sbin/sshd -D &
```

This starts sshd as the `sshdaemon` user instead of root. `gosu` handles signal forwarding correctly (unlike `su` which creates a subprocess).
