<objective>
Migrate the Angular client to the new POST contract that Plan 01 established. Extend
`RestService.post` to carry an optional JSON body (existing no-body callers unaffected),
rewrite `ConfigService.set`'s transport block to POST `{section, key, value}` as a raw JSON
body to the bare `/server/config/set` URL (deleting the `CONFIG_SET_URL` template and both
`encodeURIComponent` calls), and migrate `config.service.spec.ts` from GET-URL expectations to
POST expectations.

Purpose: The browser no longer puts config values (including credentials) into the request URL
— closing the client side of CFG-01. The Settings page saves over POST end-to-end (CFG-03).
Output: An extended `RestService.post`, a rewritten `ConfigService.set` transport block, and a
migrated spec.

Depends on Plan 01: the body shape (`{section, key, value}` bare URL) and the POST method are
the contract Plan 01 fixed on the backend. This plan must mirror it exactly.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Settings UI → ConfigService.set | User-entered config values (incl. credentials) enter here; values must not be placed into request URLs |
| ConfigService → RestService → HttpClient | The request the browser actually emits (URL + body) crosses to the network; URL must carry nothing sensitive |
| Browser → backend POST /server/config/set | Same-origin authenticated API call; auth/rate-limit enforced server-side (Plan 01 / web_app.py) |
