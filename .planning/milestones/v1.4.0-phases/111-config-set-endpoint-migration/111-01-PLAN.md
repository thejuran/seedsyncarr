<objective>
Migrate the backend `/server/config/set` HTTP contract from the credential-leaking
`GET /server/config/set/<section>/<key>/<value:re:.+>` path-segment form to a
bare `POST /server/config/set` reading a JSON body `{section, key, value}`, and **fully
remove** the legacy GET route (hard cutover — no dual-support). Migrate both test files to
the new contract, replace the two now-obsolete encoding tests with raw-value-through-body
tests, and add new coverage for the malformed-body and old-path-removal error surfaces.

This is the **leaf dependency** of Phase 111: it fixes the HTTP contract that the Angular
client (Plan 02) and the E2E setup script + Playwright page objects (Plan 03) consume. It must
land first so the downstream POST callers have a real endpoint to hit.

Purpose: Credential values (including `lftp/remote_password`) stop traveling in URLs — they no
longer land in server access logs, browser history, or reverse-proxy logs (CFG-01). The
credential-leaking GET path is gone, not deprecated-but-live (CFG-02). On-disk config format is
unchanged (CFG-04).
Output: Rewritten handler + route registration in `config.py` (with the orphaned `unquote`
import removed), and two fully-migrated test files with new error-surface coverage.
</objective>

| Browser/HTTP client → Bottle WebApp | Untrusted request input (method, path, headers, body) crosses here; auth + rate-limit enforced by `before_request` + decorator |
