<objective>
Migrate the E2E layer to the POST contract from Plan 01: rewrite all 11 `curl` config-set calls
in the E2E setup script to `POST` with a JSON body (mirroring the existing `/server/command/restart`
POST at line 34), and switch every Playwright config-set helper — the 7 in `settings.page.ts`,
`setRateLimit` in `seed-state.ts`, and the 2 inline calls in `dashboard.page.spec.ts` — to
`page.request.post('/server/config/set', { data: {section, key, value} })`. Delete the stale
"GET-only" comment and the obsolete `CONFIG_SET` GET-URL constant.

Purpose: The E2E setup script bootstraps the entire E2E suite's config state — if its config-set
calls don't move to POST, every downstream Playwright test fails against the new endpoint. This
plan is on the critical path for CFG-03's "saves end-to-end" + CI-green requirement, and removes
the last place credential-shaped values appeared in URLs (the test surface).
Output: A migrated bash setup script and four migrated E2E TypeScript surfaces; a green E2E suite.

Depends on Plan 01: the POST method, bare URL, and `{section, key, value}` body shape are the
backend contract. This plan must mirror it exactly so the seeded state actually persists.
</objective>
