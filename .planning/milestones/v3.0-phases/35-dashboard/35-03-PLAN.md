**In `file.component.ts`:**

1. Add a constant for bar width at the class level:
   ```typescript
   readonly BAR_WIDTH = 10;
   ```

2. Add the `getAsciiBar()` method (with explicit return type per lint conventions):
   ```typescript
   getAsciiBar(): string {
       const pct = Math.min(Math.max(this.file?.percentDownloaded ?? 0, 0), 100);
       const filled = Math.round((pct / 100) * this.BAR_WIDTH);
       const empty = this.BAR_WIDTH - filled;
       return '[' + '\u2588'.repeat(filled) + '\u2591'.repeat(empty) + '] ' + pct + '%';
   }
   ```
   Use Unicode escapes `\u2588` (full block) and `\u2591` (light shade) to avoid any encoding issues with the actual characters in the source file.

   **Note on OnPush safety:** The component uses `ChangeDetectionStrategy.OnPush`. The `file` `@Input()` is an Immutable.js Record — new reference on every update — so `getAsciiBar()` only re-evaluates when the input changes. This is safe.

**In `file.component.html`:**

3. Replace the entire `.size` div contents (the `<div class="progress">...</div>` block with its `<div class="progress-bar">` child) with the ASCII bar. The `.size` div structure becomes:
   ```html
   <div class="size">
       <div class="ascii-bar"
            [class.active]="file.status === ViewFile.Status.DOWNLOADING">
           {{getAsciiBar()}}
       </div>
       <div class="size_info">
           {{file.localSize | fileSize:3}} of {{file.remoteSize | fileSize:3}}
       </div>
   </div>
   ```
   Remove the entire `<div class="progress">` block including its `<div class="progress-bar">` child with all the `[class.progress-bar-animated]`, `[class.progress-bar-striped]`, `[attr.aria-valuenow]`, and `[style.width]` bindings.
