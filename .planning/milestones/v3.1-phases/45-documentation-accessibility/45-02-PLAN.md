<objective>
Add keyboard focus trap to the confirm modal so that Tab/Shift+Tab cycle only among modal buttons and Escape dismisses the modal. On close, focus returns to the element that triggered the modal.

Purpose: Users navigating with a keyboard or assistive technology must not be able to tab out of a modal into obscured background content — this is a WCAG 2.1 AA requirement (Success Criterion 2.4.3 Focus Order, 2.1.2 No Keyboard Trap).
Output: Updated ConfirmModalService with focus trap + focus restore, and expanded test suite.
</objective>

  <files>src/angular/src/app/services/utils/confirm-modal.service.ts</files>
  <action>
Modify ConfirmModalService to implement a focus trap:

1. **Save the previously focused element:** At the start of `createModal()`, save `document.activeElement as HTMLElement` to a private field `previouslyFocusedElement`.

2. **Add aria-modal and aria-labelledby:** On the modal element, add `aria-modal="true"` attribute (already has `role="dialog"`). Add `id="confirm-modal-title"` to the `<h5 class="modal-title">` in the innerHTML template and set `aria-labelledby="confirm-modal-title"` on the modal element.

3. **Focus the Cancel button on open:** After appending the modal to the DOM, call `cancelButton.focus()` (the Cancel button is the safe default — user must explicitly choose OK). Do this with a `setTimeout(() => cancelButton.focus(), 0)` so the DOM has settled.

4. **Add keydown listener for focus trap:** Add a `keydown` event listener on `this.modalElement` that:
   - On `Escape`: call `closeModal(false)` and `event.preventDefault()`.
   - On `Tab` (without Shift): if `document.activeElement === okButton`, move focus to `cancelButton`, `event.preventDefault()`. (There are only two focusable elements: cancelButton and okButton.)
   - On `Shift+Tab`: if `document.activeElement === cancelButton`, move focus to `okButton`, `event.preventDefault()`.

   Store this listener reference in a private field `keydownHandler` so it can be cleaned up.

5. **Restore focus on close:** In `destroyModal()`, after removing DOM elements, restore focus: `if (this.previouslyFocusedElement) { this.previouslyFocusedElement.focus(); this.previouslyFocusedElement = null; }`. Also remove the keydown listener and null the handler reference.

6. **Private field declarations:** Add these private fields to the class:
   - `private previouslyFocusedElement: HTMLElement = null;`
   - `private keydownHandler: ((event: KeyboardEvent) => void) | null = null;`

Do NOT change the escapeHtml method, the ConfirmModalOptions interface, or any existing button click/backdrop click handlers.
