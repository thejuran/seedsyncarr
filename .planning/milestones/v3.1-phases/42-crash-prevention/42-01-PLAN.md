  **CRASH-01: Fix propagate_exception in app_process.py**

  In the `propagate_exception` method (around line 117-126), the current code is:
  ```python
  def propagate_exception(self):
      try:
          exc = self.__exception_queue.get(block=False)
          raise exc.re_raise()
      except queue.Empty:
          pass
  ```

  The problem: `exc.re_raise()` calls `raise self.ee.with_traceback(self.tb)` inside itself, which means the exception is raised inside `re_raise()`. The outer `raise` on line 124 is redundant — if `re_raise()` succeeds in raising, the outer `raise` never executes. But the pattern is confusing and if `re_raise()` ever returned (e.g., if the wrapped exception were None), `raise None` would mask the issue with a TypeError.

  Fix: Change `raise exc.re_raise()` to just `exc.re_raise()`. This makes the control flow explicit — `re_raise()` raises the original exception, and there's no redundant outer raise that could mask the traceback.

  Add a test in test_app_process.py that verifies `propagate_exception` re-raises the original exception type (e.g., DummyException) and that the traceback is preserved. The test should start a DummyProcess(fail=True), wait for it to propagate an exception, then assert the correct exception type is raised.

  **CRASH-03: Fix bare except in webhook_manager.py**

  In the `process` method (around line 60-65), the current code is:
  ```python
  while not self.__import_queue.empty():
      try:
          source, file_name = self.__import_queue.get_nowait()
      except:
          # Queue empty (race condition between empty() and get_nowait())
          break
  ```

  The bare `except:` catches ALL exceptions, including SystemExit, KeyboardInterrupt, and programming errors (TypeError, AttributeError, etc.), masking real bugs.

  Fix: Change `except:` to `except Empty:` and add `from queue import Empty` at the top (or use the existing `Queue` import — the `Empty` exception is at `queue.Empty`). Since `Queue` is already imported from `queue`, add `Empty` to that import: `from queue import Queue, Empty`.

  Add a test in test_webhook_manager.py that verifies processing an empty queue does not raise and returns an empty list (this test already exists as `test_process_empty_queue_returns_empty` — verify it passes after the change).

  **CRASH-02: Fix _estimate_root_eta None guard**

  In `_estimate_root_eta` (around line 443-459), the method checks for `transferred_size is None` but does NOT check for `remote_size is None` before line 458:
  ```python
  remaining_size = max(model_file.remote_size - model_file.transferred_size, 0)
  ```

  If `remote_size` is None, this line crashes with `TypeError: unsupported operand type(s) for -: 'NoneType' and 'int'`.

  This can happen when a file is DOWNLOADING (has an active LFTP transfer) but the remote scanner hasn't returned size info yet (remote_size stays None from ModelFile init).

  Fix: Add a None guard for `remote_size` alongside the existing `transferred_size` guard. Insert after the `transferred_size` check:

  if model_file.remote_size is None:
