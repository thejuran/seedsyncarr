  <action>
1. In `src/python/ssh/sshcp.py`, update the `__run_command` method's SSH options (around line 52-54):
   - Change `StrictHostKeyChecking=no` to `StrictHostKeyChecking=accept-new`
   - Change `UserKnownHostsFile=/dev/null` to remove it entirely (use the default `~/.ssh/known_hosts`)
   - Update the comment from "ignore host key changes" to "accept new host keys, reject changed ones"
   - Remove the "ignore known hosts file" comment
   - The resulting common flags should be:
     ```python
     command_args += [
         "-o", "StrictHostKeyChecking=accept-new",  # accept new host keys, reject changed ones
         "-o", "LogLevel=error",  # suppress warnings
     ]
     ```

2. In `src/python/ssh/sshcp.py`, add a new pexpect expect pattern for host key verification failure. In both `expect` calls within `__run_command`, add a pattern for the host key changed warning. After the existing `'Connection timed out'` pattern, add:
   - `'REMOTE HOST IDENTIFICATION HAS CHANGED'` (i=8 in first expect, corresponding index in second)
   - When this pattern matches, raise `SshcpError("Remote host key has changed. This may indicate a MITM attack. Remove the old key from ~/.ssh/known_hosts to continue.")`

3. In `src/docker/build/docker-image/Dockerfile` (line 128), change:
   ```
   RUN mkdir -p /root/.ssh && echo "StrictHostKeyChecking no\nUserKnownHostsFile /dev/null" > /root/.ssh/config
   ```
   To:
   ```
   RUN mkdir -p /root/.ssh && echo "StrictHostKeyChecking accept-new" > /root/.ssh/config
   ```
   This allows the Docker image to accept host keys on first connection and remember them, while rejecting changed keys.

4. In `src/docker/test/python/Dockerfile` (line 20), the test environment can keep `StrictHostKeyChecking no` because it's an ephemeral test container connecting to localhost. However, update the comment to make this explicit:
   ```
   # Test-only: disable host key checking for ephemeral test container connecting to localhost sshd
   RUN echo "StrictHostKeyChecking no\nUserKnownHostsFile /dev/null\nLogLevel=quiet" > /root/.ssh/config
   ```

5. LFTP note: The `sftp:auto-confirm` setting in `src/python/lftp/lftp.py` (set to `True` at line 84) controls LFTP's own host key auto-confirmation. This is a separate mechanism from the SSH `StrictHostKeyChecking` option and is acceptable for LFTP's SFTP connections. Leave it as-is — LFTP manages its own known hosts independently.
