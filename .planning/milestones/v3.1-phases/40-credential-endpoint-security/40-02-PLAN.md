  <action>
In `delete_process.py`, add `import shlex` at the top.

In `DeleteRemoteProcess.run_once()`, change line 50 from:
```python
out = self.__ssh.shell("rm -rf '{}'".format(file_path))
```
to:
```python
out = self.__ssh.shell("rm -rf {}".format(shlex.quote(file_path)))
```

`shlex.quote()` wraps the path in single quotes AND properly escapes any embedded single quotes (e.g., `it's` becomes `'it'"'"'s'`). This is strictly more secure than the current single-quote wrapping which breaks if the file_name itself contains a single quote.

In the test file `test_file_operation_manager.py`, add tests for `DeleteRemoteProcess` shell escaping. Since `DeleteRemoteProcess` depends on `Sshcp`, mock it:

1. `test_delete_remote_escapes_single_quotes_in_filename` — Create `DeleteRemoteProcess` with file_name containing a single quote (e.g., `"it's a file"`). Mock `Sshcp.shell` via patching. Call `run_once()`. Assert the shell command passed to `Sshcp.shell` uses `shlex.quote` output (the exact escaped form).
2. `test_delete_remote_escapes_semicolons_in_filename` — file_name = `"file; rm -rf /"`, assert the shell command properly quotes it so the semicolon is not interpreted.
3. `test_delete_remote_normal_filename` — file_name = `"normal_file.txt"`, assert the command is `rm -rf '/remote/path/normal_file.txt'` (shlex.quote wraps in single quotes for safe names).
