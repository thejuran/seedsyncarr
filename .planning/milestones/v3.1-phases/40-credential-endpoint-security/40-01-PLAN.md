<task type="auto">
  <name>Task 2: Scrub passwords from SSE log stream records</name>
  <files>

  <action>
The LFTP password leak occurs because `lftp.py` line 66 passes `-u user,password` to pexpect.spawn, and when verbose logging is on (`__log_command_output = True`), the `__run_command` method at line 125 logs `command: ...` which includes the full LFTP command with password in arguments. These log records flow through `LogStreamHandler` → `SerializeLogRecord.record()` → SSE stream → browser.

In `serialize_log_record.py`, add a `_redact_sensitive` static method that takes a message string and returns a sanitized version. Use `re.sub` to find and replace password patterns:

1. LFTP `-u user,password` pattern: `re.sub(r'(-u\s+\S+,)\S+', r'\1**REDACTED**', message)` — this matches `-u username,` followed by the password, replacing only the password portion
2. As a safety net, also redact any `password=<value>` patterns: `re.sub(r'(password[=:]\s*)\S+', r'\1**REDACTED**', message, flags=re.IGNORECASE)`

Call `_redact_sensitive()` on `record.msg` before inserting into `json_dict[__KEY_MESSAGE]`. Also apply it to `exc_text` if present (exception tracebacks could also contain passwords).

Create a new test file `tests/unittests/test_web/test_handler/test_stream_log.py` with tests:
1. `test_redacts_lftp_password_from_log_message` — Create a LogRecord with msg containing `-u remoteuser,secretpass123 sftp://host`, serialize via `SerializeLogRecord().record()`, parse the SSE data JSON, assert message contains `-u remoteuser,**REDACTED**` and does NOT contain `secretpass123`
2. `test_redacts_password_equals_pattern` — LogRecord msg `"password=mysecret"` → assert `"password=**REDACTED**"` in output
3. `test_preserves_normal_messages` — A normal log message like `"Downloading file.txt"` passes through unchanged
4. `test_redacts_password_in_exception_traceback` — Set `exc_text` containing password string, verify it is redacted in output
  </action>
