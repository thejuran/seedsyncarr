**AutoQueueService.remove() stale index fix (FE-04):**

In the `remove()` method, the subscribe callback at line 108-111 uses `currentPatterns` (captured before the HTTP request on line 93) to find the index for removal, but reads `this._patterns.getValue()` for the actual list to remove from. If patterns were added/removed between the request and response, `currentPatterns.findIndex(...)` returns a stale index applied to the new list — potentially removing the wrong pattern or crashing.

Fix: Inside the subscribe success callback, read the CURRENT state for BOTH the list and the index:

```typescript
next: reaction => {
    if (reaction.success) {
        const patterns = this._patterns.getValue();
        const finalIndex = patterns.findIndex(pat => pat.pattern === pattern);
        if (finalIndex >= 0) {
            const newPatterns = patterns.remove(finalIndex);
            this._patterns.next(newPatterns);
        }
    }
}
```

Key change: `finalIndex` is now computed from `patterns` (fresh state) NOT `currentPatterns` (stale snapshot). Also add a guard `finalIndex >= 0` to handle the edge case where the pattern was already removed by another operation.

**StreamDispatchService timer cleanup (FE-05):**

StreamDispatchService creates two types of timers that are never cleaned up:
1. `this._timeoutCheckInterval = setInterval(...)` in `startTimeoutChecker()` — runs every 5 seconds forever
2. `setTimeout(() => { this.createSseObserver(); }, ...)` in `reconnectDueToTimeout()` and the error handler — reconnect timers

Fix:
1. Add `OnDestroy` to the imports from `@angular/core` and implement it on `StreamDispatchService`
2. Add a `private _reconnectTimer: ReturnType<typeof setTimeout> | null = null;` field
3. Store the setTimeout return value in `_reconnectTimer` in both `reconnectDueToTimeout()` and the error handler
4. Add ngOnDestroy method:
   ```typescript
   ngOnDestroy(): void {
       if (this._timeoutCheckInterval) {
           clearInterval(this._timeoutCheckInterval);
           this._timeoutCheckInterval = null;
       }
       if (this._reconnectTimer) {
           clearTimeout(this._reconnectTimer);
           this._reconnectTimer = null;
       }
       if (this._currentEventSource) {
           this._currentEventSource.close();
           this._currentEventSource = null;
       }
   }
   ```
5. Update the two setTimeout calls in `reconnectDueToTimeout()` and the error handler to store the timer:
   - `this._reconnectTimer = setTimeout(() => { this.createSseObserver(); }, this.STREAM_RETRY_INTERVAL_MS);`
