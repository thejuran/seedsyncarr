Refactor `sendRequest()` method to eliminate the nested subscribe anti-pattern:

1. Replace the `new Observable<WebReaction>(observer => { this._http.get(...).subscribe(...) })` pattern with a direct pipe chain:

```typescript
public sendRequest(url: string): Observable<WebReaction> {
    return this._http.get(url, {responseType: "text"}).pipe(
        map(data => {
            this._logger.debug("%s http response: %s", url, data);
            return new WebReaction(true, data, null);
        }),
        catchError((err: HttpErrorResponse) => {
            let errorMessage: string = null;
            this._logger.debug("%s error: %O", url, err);
            if (err.error instanceof Event) {
                errorMessage = err.error.type;
            } else {
                errorMessage = err.error;
            }
            return of(new WebReaction(false, null, errorMessage));
        }),
        shareReplay(1)
    );
}
```

2. Update imports to add `map`, `catchError` from `rxjs/operators` and `of` from `rxjs`.

3. Remove the `Observable` import only if it is no longer used (it will still be needed for the return type). Keep the import but verify `Observable` is still in the import list since it's the return type.

4. The behavior MUST remain identical: success returns WebReaction(true, data, null), error returns WebReaction(false, null, errorMessage), and shareReplay(1) is preserved.
