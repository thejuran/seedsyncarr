  <name>Task 1: Convert sshcp.py pexpect.spawn to argument list</name>
  <files>src/python/ssh/sshcp.py</files>
  <action>
In src/python/ssh/sshcp.py, the current pattern builds a command string via `" ".join(command_args)` and then calls `pexpect.spawn(command)` which passes the string to a shell for interpretation.

Fix: Pass the executable and argument list directly to pexpect.spawn instead of a shell string.

Current code (approximately lines 66-72):
```python
command_args.append(args)
command = " ".join(command_args)
self.logger.debug("Command: {}".format(command))
start_time = time.time()
sp = pexpect.spawn(command)
```

Change to:
```python
command_args.append(args)
self.logger.debug("Command: {}".format(command_args))
start_time = time.time()
sp = pexpect.spawn(command_args[0], command_args[1:])
```

This removes the `command = " ".join(command_args)` line entirely. The first element of command_args is the executable (e.g., "scp" or "ssh"), and the rest are arguments. pexpect.spawn with separate executable and args does NOT invoke a shell, so metacharacters in file paths are treated as literal characters.

Update the debug log to show the args list instead of a joined string for debugging clarity.

Note: command_args is built as a list throughout the method. The `args` parameter appended at the end is the SCP source/destination string or SSH command. Review the full method to ensure all items in command_args are individual arguments (they are — each `-o` flag and its value are separate list items). The `args` parameter is the final positional argument (e.g., remote path for SCP or command string for SSH shell). For SSH shell commands, the remote command is passed as a single argument to ssh which handles it correctly.
