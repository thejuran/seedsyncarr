    **CR-02 — Full focus trap:**

    The current focus trap (lines 121-135) only intercepts Tab when the active element is one of the two buttons. If focus somehow lands on another element inside the modal (e.g., the modal container itself via tabindex="-1"), Tab will escape to background content because the handler doesn't call preventDefault().

    Replace the current Tab handling logic with a version that ALWAYS traps Tab/Shift+Tab, cycling between only the two buttons:

    ```typescript
    } else if (event.key === "Tab") {
        event.preventDefault();
        if (event.shiftKey) {
            // Shift+Tab: always move to the other button
            if (document.activeElement === cancelButton) {
                okButton.focus();
            } else {
                cancelButton.focus();
            }
        } else {
            // Tab: always move to the other button
            if (document.activeElement === okButton) {
                cancelButton.focus();
            } else {
                okButton.focus();
            }
        }
    }
    ```

    This ensures:
    - Tab from Cancel -> OK (forward)
    - Tab from OK -> Cancel (wrap)
    - Shift+Tab from OK -> Cancel (backward)
    - Shift+Tab from Cancel -> OK (wrap)
    - Tab from any other element (e.g., modal backdrop) -> OK (safe default)
    - Shift+Tab from any other element -> Cancel (safe default)

    **CR-05 — Escape okBtn, cancelBtn, and btnClass:**

    Currently, `options.title` and `options.body` are escaped via `escapeHtml()` (lines 51-52), but the button text (`okBtn`, `cancelBtn`) and button CSS classes (`okBtnClass`, `cancelBtnClass`) are interpolated directly into innerHTML without escaping.

    After the existing lines 46-49 where defaults are assigned, escape all four values:

    ```typescript
    const okBtn = options.okBtn || "OK";
    const okBtnClass = options.okBtnClass || "btn btn-primary";
    const cancelBtn = options.cancelBtn || "Cancel";
    const cancelBtnClass = options.cancelBtnClass || "btn btn-secondary";

    const safeTitle = ConfirmModalService.escapeHtml(options.title);
    const safeBody = ConfirmModalService.escapeHtml(options.body);
    const safeOkBtn = ConfirmModalService.escapeHtml(okBtn);
    const safeOkBtnClass = ConfirmModalService.escapeHtml(okBtnClass);
    const safeCancelBtn = ConfirmModalService.escapeHtml(cancelBtn);
    const safeCancelBtnClass = ConfirmModalService.escapeHtml(cancelBtnClass);
    ```

    Then update the innerHTML template to use the safe variants:
    - `class="${safeCancelBtnClass}"` instead of `class="${cancelBtnClass}"`
    - `class="${safeOkBtnClass}"` instead of `class="${okBtnClass}"`
    - `>${safeCancelBtn}</button>` instead of `>${cancelBtn}</button>`
    - `>${safeOkBtn}</button>` instead of `>${okBtn}</button>`

    Note: The CSS class escaping is defense-in-depth. Currently callers pass static class strings, but a future caller might pass user input. The escaping is cheap and prevents any future injection vector.
