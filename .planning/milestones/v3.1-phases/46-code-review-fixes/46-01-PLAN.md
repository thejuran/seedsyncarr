    In `src/python/web/serialize/serialize_log_record.py`, line 57-58, change `record.msg` to `record.getMessage()`. The `getMessage()` method returns the fully interpolated message string (applying `record.args` to `record.msg`), while `record.msg` is just the format template. If a password is passed as a format argument (e.g., `logger.info("connecting with %s", password)`), `record.msg` is `"connecting with %s"` which has nothing to redact, but `getMessage()` returns `"connecting with secret123"` which does get redacted.

    In `test_stream_log.py`, add a new test `test_redacts_password_in_format_args` inside `TestSerializeLogRecordRedaction`:
    - Create a LogRecord where `msg="connecting with -u user,%s"` and `args=("secretpass123",)`:

      ```python
      record = LogRecord(
          name="test.logger", level=20, pathname="test.py", lineno=1,
          msg="connecting with -u user,%s", args=("secretpass123",), exc_info=None,
      )
      ```
    - Serialize with `SerializeLogRecord().record(record)`
    - Parse the SSE message and assert `"**REDACTED**"` is in the message
    - Assert `"secretpass123"` is NOT in the message
