<objective>
Eliminate the single raw `innerHTML` sink in `ConfirmModalService.createModal()` (BUG-01). Replace the string-template DOM build (confirm-modal.service.ts lines 100-116) with structural `Renderer2` construction — `createElement`/`createText`/`appendChild`/`addClass`/`setAttribute` — so each user-supplied string (`title`, `body`, `okBtn`, `cancelBtn`) becomes an inert DOM text node and each class string becomes a structurally-set attribute, neutralizing XSS without any escape routine. Remove the now-dead `escapeHtml()` static and the six `safe*` locals (D-02). Fold in the deferred skipCount hardening: coerce `options.skipCount` with `Number()` and gate on `Number.isFinite(n) && n > 0` (D-04), so a `toString()`-overriding object can no longer inject markup through `skipMessage`.

Purpose: BUG-01 closes a DOM-based XSS vector (CWE-79). Text-node rendering is a browser-level structural guarantee, strictly stronger than entity-escaping a string before assigning `innerHTML`.

Output: `confirm-modal.service.ts` with no `innerHTML`/`outerHTML`/`insertAdjacentHTML` assignment; updated `confirm-modal.service.spec.ts` (escapeHtml unit tests removed, mechanism-specific entity-encoding assertions converted to structural outcome assertions, D-05 runtime-boundary probe inverted).
</objective>
