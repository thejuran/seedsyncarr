<objective>
Apply the shared sanitize_log_value() helper (built in Plan 01) to the **webhook/command taint cluster** — the log sites fed directly by *arr webhook payloads and by user-supplied command file names — closing CWE-117 log injection for those sinks (D-01 + re-derived D-03).

This plan covers the cluster outside the auto-delete timer method group, so it has a clean controller.py footprint. The auto-delete-timer + model-layer cluster (which also logs remote-derived file names but spans a large method group) is split into Plan 05 (Wave 3); both plans modify controller.py, so file-ownership separation forces sequencing rather than parallel waves.
