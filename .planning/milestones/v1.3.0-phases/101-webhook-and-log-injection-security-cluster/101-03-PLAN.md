<objective>
Normalize the config GET response so secret-set vs secret-unset is indistinguishable (SEC-02, D-10/D-11, Option A — minimal). `general.webhook_secret` and `general.api_token` always serialize as `""` on BOTH the authenticated and unauthenticated paths — replacing both the current `**REDACTED**` mask (unauthenticated) and the real-value emission (authenticated).

Research clarified that no `*_is_set` boolean exists today and no Angular code reads these values from the GET response (RESEARCH.md A2). Option A (zero the value field) is the literal implementation of D-10/D-11; Option B (additive `*_is_set` booleans) is explicitly NOT chosen.

COMPAT: the SET path (`__handle_set_config`), on-disk persist format, and `_SECRET_FIELD_PATHS` are untouched. The only behavior change is the GET value for these two fields (authenticated path now returns `""` instead of the real value — a documented, intentional change under D-10's authority; RESEARCH.md Pitfall 5).

Purpose: Eliminate the secret-presence oracle / information-disclosure surface in the config GET response.
Output: `_ALWAYS_BLANK_FIELDS` loop in serialize_config.py; updated existing redaction test; new always-blank tests at both the serializer and handler layers.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-101-08 | Information Disclosure | serialize_config.py (secret-presence oracle / value leak, SEC-02) | mitigate | Always serialize webhook_secret and api_token as "" on both auth paths; set/unset are indistinguishable in value, length, and key presence (D-10/D-11). Test asserts no sentinel value appears in the payload. |
| T-101-09 | Information Disclosure (INTRODUCED) | breaking the authenticated operator config-edit flow (Pitfall 5) | accept | Authenticated GET no longer returns the real secret value. This is an intentional behavior change under D-10's authority; SET path is untouched so operators still write secrets normally. Low impact, accepted. |
| T-101-10 | Tampering (INTRODUCED) | accidentally blanking non-secret fields or touching the SET path | mitigate | _ALWAYS_BLANK_FIELDS scoped to exactly general.webhook_secret + general.api_token; __handle_set_config, on-disk format, and _SECRET_FIELD_PATHS untouched. Existing serializer tests for lftp/sonarr/radarr redaction stay green, pinning no collateral change. |
