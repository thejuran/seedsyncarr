<objective>
Close the remaining SEC-01 log-injection sinks that Plan 04 handed off: the **auto-delete timer cluster** in controller.py and the **model-layer remote-scanner** debug logs in model/model.py. These all interpolate file names that originate from the seedbox remote directory listing (lftp scan output) — the same remote-influenceable taint class as the sites Plan 04 sanitized — and currently log them raw.

BLOCKER-4 re-evaluation of the previously-deferred controller.py:229: it logs a key from `__pending_auto_deletes` (the exit-time timer-cancel loop). That dict is populated at controller.py:817 inside `__schedule_auto_delete`, whose `file_name` argument is the remote-derived `root_name` passed from `__check_webhook_imports` (controller.py:803-804). So :229 logs the SAME scheduled/remote-derived name as :820 — deferring it is unsound. It is sanitized here, NOT deferred.
