<objective>
Create the shared `sanitize_log_value()` log-injection helper (SEC-01, CWE-117) in `common/types.py` and re-export it from `common/__init__.py`. This is the leaf dependency: Plans 04 and 05 replace the three inline `.replace("\n","\\n").replace("\r","\\r")` copies and add the remaining call sites — all depend on this helper existing first.

Per REQUIREMENTS.md SEC-01 and CONTEXT D-01, the helper escapes **CR/LF AND control characters** — not CR/LF alone. CR and LF render as the existing literal `\r`/`\n` escapes (continuity with the inline copies being replaced); every other C0 control character (0x00–0x1F, which includes ESC 0x1B used in ANSI-escape injection, and TAB 0x09) plus DEL (0x7F) is neutralized via a `\xHH` hex escape. C1 controls (0x80–0x9F) are intentionally left unescaped (documented below) to avoid corrupting legitimate multi-byte UTF-8 filename content; the threat model (ANSI/CRLF log forging) is covered by C0+DEL.

This plan ONLY creates the helper and its unit tests. It does NOT change any call site (that is Plan 04/05). Production log output is unchanged until those plans run, so this plan is behavior-neutral on its own.

Purpose: One definition, no copy-paste drift, single source of truth for CWE-117 escaping covering the full control-character set (D-01).
Output: `sanitize_log_value()` in `common/types.py`, re-export in `common/__init__.py`, `TestSanitizeLogValue` unit tests.
</objective>
