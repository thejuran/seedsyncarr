`controller/model_pipeline.py` collaborator. This is plan 3 of 3 (D-06 — the largest seam, done last when
`controller.py` is already thinned by plans 01 and 02). It is the bulk of the line count
(`controller.py:342-655`).

`_collect_extract_results`, `_set_import_status`, `_feed_model_builder`, `_detect_and_track_queued`,
`_detect_and_track_download`, `_prune_extracted_files`, `_prune_downloaded_files`, `_apply_model_diff`,
`_build_and_apply_model`. `__update_model` (`:708-743`) thins to delegate to
`self.__model_pipeline.update_model()` plus the retained coordinator stages.

and 2, 109-PATTERNS.md section "What stays on Controller"):

- `_should_update_capacity` (`@staticmethod`, `:657-669`) — tests call `Controller._should_update_capacity(...)`.
- `_update_controller_status` (`:671-706`) — calls `Controller._should_update_capacity(...)`; keeping it on
  the coordinator avoids a Controller to ModelPipeline to Controller cycle.
- `_update_active_file_tracking` (`:402-426`) — writes `self.__active_downloading_file_names`, which tests
  read via `_Controller__active_downloading_file_names`.
- All model-read accessors (`get_model_files`, `get_model_files_and_add_listener`,
  `add/remove_model_listener`, `is_file_stopped`, `is_file_downloaded`, `__get_model_files`) — the web
  layer's frozen public surface.
