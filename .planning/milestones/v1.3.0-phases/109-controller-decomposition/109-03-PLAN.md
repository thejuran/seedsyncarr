<objective>
Extract the scan->build->diff->apply model-update pipeline LOGIC out of `Controller` into a new
`controller/model_pipeline.py` collaborator. This is plan 3 of 3 (D-06 — the largest seam, done last when
`controller.py` is already thinned by plans 01 and 02). It is the bulk of the line count
(`controller.py:342-655`).

The pipeline-stage BODIES move into a `ModelPipeline` class. **CRITICAL — test-pinned helpers (BLOCKER from
adversarial review):** the EXISTING, UNMODIFIED unit suite calls nine of these stage methods DIRECTLY on a
`Controller` instance as `self.controller._method(...)` (single-underscore, NOT `_Controller__` mangled).

("the full pre-refactor Python test suite passes WITHOUT modification or deletion"). Therefore, for EACH
test-pinned helper, **only the body moves** — `Controller` KEEPS a `def _method(self, ...)` whose body is a
one-line forwarding delegation to `self.__model_pipeline.<method>(...)`. The callable name never leaves
Controller; only the logic relocates.
