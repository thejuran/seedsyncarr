<objective>
Close BUG-03 (auto-delete callback race on shutdown) by adding a dedicated `threading.Event`
shutdown guard so an already-fired auto-delete callback that is mid-execution when `exit()` runs
no-ops cleanly — performing no model read and dispatching no `delete_local`.

Purpose: Criterion #1 (timers tracked + cancelled on exit, none armed afterward) is ALREADY
SATISFIED — it landed in Phase 101 Plan 05 (`__pending_auto_deletes` dict, `__auto_delete_lock`,
cancel-all-on-exit). Per D-01 this plan VERIFIES that with a test; the production change for #1 is
a no-op. The real work is criterion #2 (D-02): a callback that fired a microsecond before `exit()`
must not act against a half-torn-down model. A dedicated Event (not the `__started` bool, per D-03;
no thread drain/join, per D-04) is the minimal correct fix.

Output:
- A new `self.__shutdown_event = threading.Event()` in the controller, set in `exit()` before the
  cancel loop, and checked at `__execute_auto_delete` entry under `__auto_delete_lock`.
- A test-first red→green shutdown-guard test plus (if absent) a criterion-#1 verification test,
  both added to the existing `TestAutoDeleteToggleDuringTimer` harness.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| remote model → controller | file_name and model metadata originate from the remote LFTP/scan side; any value logged on the auto-delete path is remote-derived and untrusted as log content |
| timer thread → main thread (exit) | a fired auto-delete callback runs on a daemon Timer thread concurrently with exit() running on the main thread — a TOCTOU / race boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-102-01 | Tampering / Denial of Service | __execute_auto_delete firing during/after exit() against a half-torn-down model | mitigate | This IS the BUG-03 fix: __shutdown_event checked at execute entry under __auto_delete_lock causes the callback to no-op (no model read, no delete_local) once shutdown has been signalled. |
| T-102-02 | Information Disclosure | log line on the guarded-out path interpolating remote-derived file_name | mitigate | Wrap file_name with sanitize_log_value() per controller.py convention (CONVENTIONS.md) — no raw remote string reaches the log. |
| T-102-03 | Tampering | npm/pip/cargo installs | accept | No new dependencies introduced; threading and pytest are already in the project. No install task in this plan, so the supply-chain surface is unchanged. |
