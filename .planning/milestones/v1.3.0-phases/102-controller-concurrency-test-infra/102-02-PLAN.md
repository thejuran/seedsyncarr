<objective>
Close INFRA-01: make the multiprocessing-logger unit tests exercise the `spawn` start method so the
spawn code path is covered on Linux CI (where the default is `fork`), and promote each test's local
`process_1` closure to one shared, picklable, module-level helper so it survives the spawn pickle.

Purpose: Under `fork`, a test-local closure is inherited by the child and "works"; under `spawn` the
child re-imports the module and unpickles the target, so a local closure cannot be the Process target.
This refactor (D-06 + D-07) makes the three tests robust under spawn AND verifies the spawn path on CI.
It is strictly the LOWEST priority item and MUST NOT expand the milestone (D-07/D-08).

Output:
- One module-level, parametrized, picklable helper (e.g. `_emit_log_records(mp_logger, spec)`).
- The three tests (test_main_logger_receives_records ~215, test_children_names ~245, test_logger_levels
  ~270) each use `multiprocessing.get_context('spawn')` + `ctx.Process(target=_emit_log_records, args=...)`.
- The production module and every `log_capture.check(...)` assertion remain unchanged.
</objective>
