<design_note>
## Sanctioned Phosphor → FA4.7 icon fallback (Delete Remote only)

**Context:** Per user design-spec-rigor memory ("Port AIDesigner HTML identically, no approximations"), every Variant B value is ported literal — hex, rgba, padding, border-radius, font-size, font-weight, class name. The Variant B HTML uses Phosphor icons; per CONTEXT.md (`.planning/phases/72-restore-dashboard-file-selection-and-action-bar/72-CONTEXT.md` line 87), the project rule is "text-only UI — Variant B uses Phosphor icons in small roles, map to equivalent Font Awesome 4.7 glyphs per memory." This plan honors that sanctioned mapping rule for ALL 5 icons (play / stop / file-archive-o / trash / cloud+ban).

**Single icon deviation — Delete Remote:** Phosphor ships `ph-cloud-slash` as a single-glyph "cloud with strike-through" icon. FA 4.7 has NO equivalent single-glyph — no `fa-cloud-slash`, no `fa-cloud-off`. The closest approximation is stacking `<i class="fa fa-cloud">` + `<i class="fa fa-ban action-overlay">` where the `.action-overlay` class positions the ban glyph over the cloud via `margin-left: -0.9em` (SCSS rule in Task 2). This is NOT a palette deviation (the hex values remain Variant B literal — `#c97d7d` status-red for the ban overlay color) — only the icon family is mapped, which the CONTEXT.md rule explicitly permits.

**Scope of this fallback:**
- Applies ONLY to the Delete Remote button's icon construction.
- The other 4 action-button icons (Queue / Stop / Extract / Delete Local) each have a single-glyph FA4.7 equivalent (`fa-play`, `fa-stop`, `fa-file-archive-o`, `fa-trash`) — no stacking, no approximation.
- All palette hex values, paddings, gaps, border-radii, font sizes, and class names are ported literally from Variant B — no approximations, no shortcuts.

**Downstream consequence for the 72-01-SUMMARY.md write step:** the SUMMARY MUST restate this design note verbatim under an `## Icon mapping` section so the checker/auditor can trace the sole deviation to its sanction (CONTEXT.md line 87) without re-deriving context. This design_note inside the PLAN serves as the executor's explicit permission — the executor MUST NOT swap to an alternative single-glyph under "rigor" pressure, because NO alternative single-glyph exists in FA4.7.
</design_note>
