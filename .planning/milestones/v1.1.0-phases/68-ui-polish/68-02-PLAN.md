```html

    <span class="version-badge">Version {{ version }} (Stable)</span>
```

In `about-page.component.scss`, add a `.version-link` rule near the `.version-badge` block:

```scss
.version-link {
    text-decoration: none;
    transition: opacity 200ms;

    &:hover {
        opacity: 0.8;
    }

    &:hover .version-badge {
        border-color: $amber;
    }
}
```

**Nav bar** -- Wrap the brand-version span in an anchor. In `app.component.html`, change:

Before:
```html
<span class="brand-version">v{{version}}</span>
```

After:
```html
<a href="https://github.com/thejuran/seedsyncarr/releases" target="_blank" rel="noopener noreferrer" class="brand-version-link">
    <span class="brand-version">v{{version}}</span>
</a>
```

In `app.component.scss`, add a `.brand-version-link` rule near the `.brand-version` block:

```scss
.brand-version-link {
    text-decoration: none;
    transition: opacity 0.15s ease;

    &:hover {
        opacity: 0.8;
    }

    &:hover .brand-version {
        border-color: $amber;
        color: $text;
    }
}
```

Note: The `app.component.scss` currently uses hardcoded hex in `.brand-version` (`color: #9aaa8a`, `background: #2c3629`, `border: 1px solid #3e4a38`). These are the same as shared aliases `$textsec`, `$muted`, `$border`. However, since plan 01 does NOT modify app.component.scss palette values (that was not in scope for the palette drift task which targeted settings+logs), use the hex values consistently with the existing file OR use the shared aliases if they are available via the existing `@use '../../common/common' as *` import. Since that import IS already present at line 1, you CAN use `$amber` and `$text` in the new hover rule -- but do NOT refactor existing hex values in this plan (that would be scope creep).
