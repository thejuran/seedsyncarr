# Requirements: SeedSync

**Defined:** 2026-02-16
**Core Value:** Reliable file sync from seedbox to local with automated media library integration

## v3.0 Requirements

Requirements for v3.0 Terminal UI Overhaul. Each maps to roadmap phases.

### Visual Identity

- [x] **VIS-01**: User sees Fira Code font for all data displays (filenames, speeds, sizes, progress)
- [x] **VIS-02**: User sees IBM Plex Sans for UI labels, buttons, and navigation
- [x] **VIS-03**: User sees deep dark backgrounds (#0d1117 base) with green accent palette (#00ff41 neon, #3fb950 readable, #238636 muted)
- [x] **VIS-04**: User sees CRT scan-line overlay effect (subtle, low opacity repeating gradient)
- [x] **VIS-05**: User sees custom dark scrollbar styling (webkit + Firefox)

### Navigation

- [x] **NAV-01**: User sees sidebar as 56px icon rail that expands to 200px on hover (CSS-only transition)
- [x] **NAV-02**: User sees `>` prompt indicator on active route in sidebar
- [x] **NAV-03**: User sees app version at bottom of sidebar
- [x] **NAV-04**: User can navigate via mobile hamburger menu (preserved from current behavior)

### File Dashboard

- [x] **DASH-01**: User sees search input with terminal prompt `>` prefix
- [x] **DASH-02**: User sees colored left border on file rows by status (green=downloading, teal=downloaded, amber=queued, red=stopped)
- [x] **DASH-03**: User sees ASCII-style block progress bars (`[████░░░░] 67%`) replacing Bootstrap progress component
- [x] **DASH-04**: User sees green glow effect on actively downloading rows (box-shadow pulse)
- [x] **DASH-05**: User sees colored dot + text for file status (no SVG icons)
- [x] **DASH-06**: User sees ghost-style action buttons with green/red outlines and glow on hover

### Secondary Pages

- [x] **PAGE-01**: User sees terminal-style section headers in Settings (`--- Server ---`)
- [x] **PAGE-02**: User sees monospace patterns in AutoQueue with green/red buttons
- [x] **PAGE-03**: User sees true terminal-style Logs (monospace, colored by level green/amber/red, no background blocks)
- [x] **PAGE-04**: User sees ASCII-art inspired About page with monospace version display

### Theme Cleanup

- [x] **CLEAN-01**: Theme toggle removed from Settings page (Appearance section removed)
- [x] **CLEAN-02**: ThemeService simplified to dark-only (no light/auto modes, no OS detection, no localStorage toggle)

## Future Requirements

### Additional *arr Support

- **ARR-01**: Lidarr integration following same *arr pattern
- **ARR-02**: Readarr integration following same *arr pattern

## Out of Scope

| Feature | Reason |
|---------|--------|
| Lidarr/Readarr integration | Separate milestone — different feature area |
| E2E tests (Playwright) | Separate concern |
| Bootstrap @import → @use migration | Blocked until Bootstrap 6 |
| Light mode preservation | Intentionally removed — Terminal/Hacker is dark-only |
| Custom animations beyond spec | Scope creep — stick to design spec |
