**Purpose:** This file is the committed "before" anchor for the **Phase 100 coverage
ratchet (RATCHET-02)**. It records main-HEAD Python and Angular coverage captured at the
start of Phase 97, *before any v1.3.0 test work lands*, so the final ratchet diffs against a
fixed reference rather than against whatever number happens to be in CI config at ratchet
time (decision D-06). The ratchet is **monotonic — thresholds only increase.**

> **PROVISIONAL — host-only, excludes the real-lftp integration suite.**
> This number was produced by the host `make coverage-python` target, which is the **only**
> existing make target that passes `--cov` (`Makefile:138-139`). The containerized
> `make run-tests-python` target **cannot** be used as the coverage source: its container CMD
> (`src/docker/test/python/Dockerfile:60`) is `["pytest", "-v", "-p", "no:cacheprovider"]` —
> there is **no `--cov` flag**, so it emits no coverage report. The host run omits the
> `tests/integration/test_lftp/*` suite (no `lftp` binary / `sshd` on the host); those tests
> errored/were skipped during this measurement and their covered lines are therefore **not**
> reflected in 85.19%. The true container-inclusive number is expected to be **higher**.
> Phase 100 MUST re-measure (see "Phase 100 follow-up" below) before setting the ratchet
> threshold.

## Phase 100 follow-up (required before ratcheting)

Phase 100 (Plan 100-03, RATCHET-02) MUST, before setting any ratchet threshold:

1. **Re-measure a container-inclusive Python "before"** using the **same `--cov` flags**
   but with the **real-lftp integration suite included** (e.g. run
   `pytest --cov --cov-report=term-missing` *inside* the test container, which has the `lftp`
   binary and `sshd`). The host number above (85.19%) is provisional and excludes that suite;
   the container-inclusive total is the honest "before" the ratchet must diff against.
2. **Create the Karma global thresholds** (`coverageReporter.check.global` in
   `karma.conf.js`) — none exist today — set at or below the measured Angular numbers, then
   ratchet upward.
3. **Ratchet `--cov-fail-under` (pytest)** upward from the current floor of `84`.

The ratchet is monotonic: thresholds may only increase from the numbers recorded here.
