  Please install the package. [plugin angular-compiler]
On Angular 22, @angular/build no longer transitively provides istanbul-lib-instrument. On main (v21) it is present in package-lock.json as a transitive dep at istanbul-lib-instrument@6.0.3 (confirmed: src/angular/package-lock.json line 380 -> "istanbul-lib-instrument": "6.0.3"). Fix: add it as an explicit devDependency pinned with caret range "^6.0.3".

Why coverage is required: the Angular unit-test job runs `ng test ... --code-coverage` (src/docker/test/angular/Dockerfile CMD). With coverage on, ng needs istanbul-lib-instrument at instrumentation time, so a transitive-only resolution that drops on v22 breaks the build.

Install-flag consistency: src/angular/.npmrc sets `legacy-peer-deps=true`, so a plain `npm install` run from inside src/angular automatically applies --legacy-peer-deps (matching CI's `npm ci` and the Docker build's `npm install --include=dev --legacy-peer-deps`). No extra flag needed, but passing --legacy-peer-deps explicitly is harmless and safe.

Conflict expectation: #51/#49 touch /src/python; #50 touches /src/angular. Different ecosystems -> no file overlap -> merging #51/#49 first will NOT make #50 out of date. Only merge main into the dependabot branch if GitHub reports it BEHIND/out-of-date after the python merges.
