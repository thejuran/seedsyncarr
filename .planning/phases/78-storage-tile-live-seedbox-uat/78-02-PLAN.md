Tile capacity-mode DOM shape (stats-strip.component.html:10-41 for Remote, :51-82 for Local):
- Main stat-value: integer % (e.g. "0%", "50%", "95%") — Angular `number:'1.0-0'` pipe, D-05 verbatim.

- Progress bar width: `used / total * 100` as a % — D-03.
- Progress bar class:
  - `stat-progress-fill--amber` when pct < 80 (Remote base)
  - `stat-progress-fill--secondary` when pct < 80 (Local base)
  - `stat-progress-fill--warning` when pct >= 80 && pct < 95
  - `stat-progress-fill--danger`  when pct >= 95
- Sub-line under bar: `"X.XX MB used"` — FileSizePipe at 2-decimal + literal " used", D-07.

Fallback-mode DOM (else branch of the @if):
- The pre-phase-74 tracked-bytes layout — NO percentage, NO "of X.XX MB", NO progress bar driven by capacity.
- Visual anchor: "X.XX MB tracked" sub-line (approximate; copy from a live fallback render in Test 3 for exact text).

Expected WARN log lines (grep targets for Test 3 evidence):
- Remote SshcpError branch:   `df SSH call failed for '/data': <err>`          (remote_scanner.py:136)
- Remote parse-failure branch:`df output parse failed for '/data': <repr>`     (remote_scanner.py:133)
- Local OSError/ValueError:   `Local disk_usage failed for path '<path>': <err>` (local_scanner.py:39)

Capacity-gate semantics (controller.py:_should_update_capacity, lines 638-650):
- new=None → returns False → no write → tile keeps showing last-known capacity until fallback is triggered by a later scan cycle.
- |new-old|/|old| ≤ 0.01 → returns False → no SSE push.
- Strict >1% diff → returns True → SSE push.

fallocate / tmpfs math (100 MiB tmpfs = 104857600 bytes):
- `fallocate -l 50M /data/fill.img` inside container → df-used ≈ 52428800 → tile shows 50% + amber.
- `fallocate -l 79M` → 79% → amber.
- `fallocate -l 80M` → 80% → warning (boundary, D-09 strict-inequality flips here).
- `fallocate -l 94M` → 94% → warning.
- `fallocate -l 95M` → 95% → danger.
- `rm /data/fill.img` → 0% → amber. (Use `fallocate -l <N>M /data/fill.img` — the prior call's allocation is replaced, not added, when the same filename is reused with a new size.)
