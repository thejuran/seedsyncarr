
with:
```python
os.chmod(TestLftp.temp_dir, 0o750)
```

**PYFIX-07** (per D-03 -- fixture approach): In `conftest.py`, find the `test_logger` fixture (around line 20). The current fixture:
1. Creates logger, adds handler, sets level to DEBUG, yields, removes handler
2. Does NOT prevent propagation, does NOT reset level, does NOT restore propagation

Apply these changes:
- Add `logger.propagate = False` BEFORE the `yield logger` line (prevents double-logging to root logger during test)
- After the existing `logger.removeHandler(handler)` line, add:
  ```python

  logger.propagate = True
  ```

The full fixture after the fix should be:
```python
@pytest.fixture
def test_logger(request):
    logger = logging.getLogger(request.node.name)
    handler = logging.StreamHandler(sys.stdout)

        logging.Formatter("%(asctime)s - %(levelname)s - %(name)s - %(message)s")
    )
    logger.addHandler(handler)
    logger.setLevel(logging.DEBUG)
    logger.propagate = False
    yield logger
    logger.removeHandler(handler)
    logger.setLevel(logging.NOTSET)
    logger.propagate = True
```

Note: conftest.py has TWO fixtures that use loggers. Only modify the `test_logger` fixture (around line 20). If there is a second logger fixture (around line 33), apply the same `propagate = False` / reset pattern to it as well to keep them consistent.
