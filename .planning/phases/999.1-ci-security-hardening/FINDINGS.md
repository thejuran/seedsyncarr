## Critical

### SEC-06: Overly broad workflow-level permissions (score: 98)
- **File:** `.github/workflows/ci.yml:15-17`
- **Issue:** `contents: write` and `packages: write` granted at workflow level; lint/unit test jobs inherit write access. A compromised dependency during `npm ci` or `pip install` could push code or images.
- **Fix:**
  ```diff
  - permissions:
  -   contents: write
  -   packages: write
  + permissions:
  +   contents: read
  ```
  Then add per-job `permissions:` blocks only where needed (`publish-docker-image` gets `packages: write`, `publish-github-release` gets `contents: write`).

## Warning

### SEC-07: GitHub Actions use floating tag versions (score: 92)
- **File:** `.github/workflows/ci.yml` (multiple lines)
- **Issue:** `actions/checkout@v4`, `actions/setup-python@v5`, `actions/setup-node@v4` are mutable tags. A compromised or hijacked action would run before any project code, with full access to the repo token.
- **Fix:**
  ```diff
  - uses: actions/checkout@v4
  + uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683 # v4
  - uses: actions/setup-python@v5
  + uses: actions/setup-python@a26af69be951a213d495a4c3e4e4022e16d87065 # v5
  - uses: actions/setup-node@v4
  + uses: actions/setup-node@49933ea5288caeca8642d1e84afbd3f7d6820020 # v4
  ```

### BUG-13: publish-github-release independent of publish-docker-image (score: 85)
- **File:** `.github/workflows/ci.yml:192-196`
- **Issue:** Both publish jobs run in parallel after E2E. If Docker push fails, a release is created pointing to a non-existent image.
- **Fix:**
  ```diff
  publish-github-release:
  -   needs: [ e2etests-docker-image ]
  +   needs: [ e2etests-docker-image, publish-docker-image ]
  ```
