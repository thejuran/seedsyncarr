<objective>
Close COVLOW-01: add a targeted regression test class proving that the auto-delete callback honors a config toggle (`enabled` or `dry_run`) flipped DURING a live `threading.Timer` window — i.e. after `__schedule_auto_delete` arms the timer but before the timer's callback re-reads config — and prove the control direction (no flip -> deletion still fires) so the negative assertions are non-vacuous.

Purpose: The existing `TestAutoDeleteExecution` tests (`test_execute_disabled_skips_deletion`, `test_execute_dry_run_does_not_delete`) flip config and then call `__execute_auto_delete` DIRECTLY. That proves the method honors config but does NOT close the COVLOW-01 gap, which is specifically the schedule -> flip -> fire path. This plan adds the missing live-Timer coverage so a future refactor that moves the config re-read out of the timer callback would be caught. Crucially, correctness is made DETERMINISTIC via a `threading.Event` handshake (not a wall-clock delay race) so the test is reliable on slow/preempted CI runners (amd64 + arm64).

Output: A new `TestAutoDeleteToggleDuringTimer` class in `test_auto_delete.py` with THREE regression tests (two negative toggle tests + one positive control). No production-code change is anticipated (the in-method re-read at controller.py:838-851 already exists and is correct) — this is a pure additive regression net.
