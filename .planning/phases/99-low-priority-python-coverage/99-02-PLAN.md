<objective>
Close COVLOW-02: add a targeted regression test proving `BoundedOrderedSet` eviction ORDER after `touch` — a touched item is retained and the oldest NON-touched item evicts first.

Purpose: The existing suite has `test_from_iterable_with_eviction` and `test_eviction_when_full` but NO test that combines `touch` with eviction ordering. Without it, a future refactor that, say, called `touch` after `add` (a CONCERNS.md risk note), or replaced `move_to_end` / `popitem(last=False)`, could silently change which item evicts. This test pins the exact LRU-after-touch behavior.

Spec-refinement note (D-03 / F5): the locked design spec §"Phase 99" wording says "touch a MIDDLE item," but this plan intentionally touches the OLDEST item ('a') per D-03. This is a deliberate, stronger refinement of the spec's wording — not a silent deviation: touching the oldest forces `move_to_end` to actually change eviction order (the oldest would otherwise have evicted first), so 'b' evicting instead of 'a' is direct proof the touch took effect. The asserted values (evicted == 'b', as_list() == ['c','a','d'], total_evictions == 1) are unchanged and correct.

Output: One new self-contained method `test_eviction_order_after_touch` on the existing `TestBoundedOrderedSet` class. No production-code change is anticipated — `touch` (move_to_end, bounded_ordered_set.py:104) and `add` eviction (popitem(last=False), bounded_ordered_set.py:85) are already correct; this is a pure additive regression net.
</objective>
