Clean up docstrings and inline comments across the Python source tree. Apply these rules:

**Rule 1 — Strip bare `:param name:` and `:return:` lines:** In every Python docstring under src/python/, delete `:param name:` lines where the description is empty or just restates the parameter name. Delete `:return:` lines where the description is empty. If the remaining docstring has only a one-line description that restates the function name (e.g., `"Adds a listener"` for `add_listener()`), delete the entire docstring. If the one-line description says something non-obvious (e.g., mentions thread-safety, side effects, concurrency), keep it.

Files with the highest density of this pattern: `model/model.py`, `lftp/lftp.py`, `web/handler/server.py`, `web/handler/controller.py`, `web/web_app.py`, `controller/controller.py`, `ssh/sshcp.py`. Scan ALL Python files under src/python/ (excluding tests/) for this pattern.

**Rule 2 — Rename "Phase N" step labels:** In `web/handler/controller.py`, change:
- `# Phase 1: Queue all commands (parallel queuing)` to `# Queue all commands (parallel queuing)`
- `# Phase 2: Wait for all callbacks to complete` to `# Wait for all callbacks to complete`

**Rule 3 — Prune trivially obvious inline comments:** In `web/handler/controller.py` `_execute_bulk_action` method, remove comments that restate what the code line does:
- `# Deduplicate files while preserving order` (before a dict.fromkeys call) — REMOVE
- `# Calculate remaining time for each callback` (before a max() call) — REMOVE
- KEEP comments that explain intent or security reasoning, e.g., `# Enforce maximum file limit to prevent DoS` — the DoS reasoning is non-obvious, so keep but simplify to `# Prevent DoS — enforce file limit`

After all changes, run `python3 -m ruff check src/python` to confirm zero violations.
