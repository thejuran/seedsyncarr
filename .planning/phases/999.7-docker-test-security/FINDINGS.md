## Warning

### SEC-03: Test user added to root group (score: 97)
- **File:** `src/docker/test/python/Dockerfile:35`
- **Issue:** `seedsyncarrtest` added to root group via `usermod -a -G root`. Grants read access to root-owned files; combined with SSH access, this is a privilege escalation path.
- **Fix:**
  ```diff
  - RUN usermod -a -G root seedsyncarrtest
  ```
  If specific directory access is needed, create a dedicated group.

### SEC-01: Hardcoded SSH password in remote test container (score: 95)
- **File:** `src/docker/test/e2e/remote/Dockerfile:14`
- **Issue:** `remoteuser:remotepass` set via `chpasswd`. Public key auth already configured; password serves no purpose.
- **Fix:**
  ```diff
  - echo "remoteuser:remotepass" | chpasswd
  + passwd -l remoteuser
  ```
  Also add `PasswordAuthentication no` to sshd_config.

### SEC-02: Hardcoded SSH password in Python test container (score: 95)
- **File:** `src/docker/test/python/Dockerfile:34`
- **Issue:** `seedsyncarrtest:seedsyncarrpass` with `PasswordAuthentication yes` explicitly enabled. Any process on the Docker network can brute-force.
- **Fix:** Lock password, switch to `PasswordAuthentication no`.

### SEC-04: Test containers run sshd as root (score: 90)
- **Files:** `src/docker/test/e2e/remote/Dockerfile:36`, `src/docker/test/python/Dockerfile:51`
- **Issue:** Last `USER` directive before `CMD` is root. Container breakout yields root-level access.
- **Fix:** Use `setcap 'cap_net_bind_service=+ep' /usr/sbin/sshd` and run as non-root, or use an unprivileged port.
