**PR #4: pytest 7.4.4 -> 9.0.3**

Known breaking changes in pytest 8+/9+:
- `pytest.raises(match=...)` no longer accepts bytes
- `pytest.warns` matches full warning message by default
- `py.path.local` replaced by `pathlib.Path`
- `nose`-style test classes deprecated
- `--strict` renamed to `--strict-markers`
- Some deprecated APIs removed

Steps:
1. `gh pr checkout 4` to get the PR branch
2. Merge main into the branch: `git merge main` (resolve poetry.lock conflicts by running `cd src/python && poetry lock --no-update`)
3. Run the Python tests locally to see what breaks:
   - `cd src/python && poetry install && poetry run pytest tests/ -x --timeout=60 2>&1 | head -100`
4. If tests pass: push, wait for CI, merge
5. If tests fail: fix the test code to be compatible with pytest 9, commit fixes to the PR branch, push, wait for CI, merge
6. Common fixes needed:
   - Replace any `py.path.local` usage with `pathlib.Path`
   - Update any deprecated fixture patterns
   - Adjust `filterwarnings` if deprecation warnings changed

**PR #6: testfixtures 10.0.0 -> 11.0.0**

Known breaking changes in testfixtures 11:
- `compare()` function signature changes

- `LogCapture` API changes possible
