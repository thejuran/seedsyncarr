<task type="auto">
  <name>Task 1: Enable GitHub Discussions and set repository topics</name>
  <files></files>
  <read_first>.planning/phases/57-readme-community-health/57-RESEARCH.md</read_first>
  <action>
1. Ensure Plan 01 changes are committed and pushed to `main` on GitHub. If not pushed yet, run `git push origin main`.

2. Enable GitHub Discussions:
```bash
gh api --method PATCH repos/thejuran/seedsyncarr --field has_discussions=true
```
Note: Use `--field` (not `-f`) for boolean values to ensure proper type coercion.

3. Verify Discussions is enabled:
```bash
gh repo view thejuran/seedsyncarr --json hasDiscussionsEnabled --jq '.hasDiscussionsEnabled'
```
Expected output: `true`

4. Verify Q&A category exists (auto-created by GitHub):
```bash
gh api graphql -f query='{
  repository(owner: "thejuran", name: "seedsyncarr") {
    discussionCategories(first: 10) {
      nodes { name }
    }
  }
}' --jq '.data.repository.discussionCategories.nodes[].name'
```
Expected output should include "Q&A" in the list.

5. Set repository topic tags (all 10 tags including the 7 required ones):
```bash
gh api --method PUT repos/thejuran/seedsyncarr/topics \
  -H "Accept: application/vnd.github.mercy-preview+json" \
  -f names[]="seedbox" \
  -f names[]="sonarr" \
  -f names[]="radarr" \
  -f names[]="lftp" \
  -f names[]="arr" \
  -f names[]="self-hosted" \
  -f names[]="file-sync" \
  -f names[]="python" \
  -f names[]="angular" \
  -f names[]="docker"
```

6. Verify topics are set:
```bash
gh api repos/thejuran/seedsyncarr/topics --jq '.names[]'
```
Expected: all 10 topics listed.
  </action>
  <verify>
    <automated>gh repo view thejuran/seedsyncarr --json hasDiscussionsEnabled --jq '.hasDiscussionsEnabled' | grep -q "true" && gh api repos/thejuran/seedsyncarr/topics --jq '.names | length' | grep -qE "^(7|8|9|10)$" && echo "PASS"</automated>
  </verify>
  <acceptance_criteria>

    - `gh api graphql` query confirms Q&A category exists in discussion categories
  </acceptance_criteria>
