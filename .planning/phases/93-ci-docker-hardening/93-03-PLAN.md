Rewrite `src/docker/test/e2e/remote/Dockerfile` with the following complete content. The changes are extensive enough that a full rewrite is clearer than incremental edits:

```dockerfile
FROM ubuntu:22.04

# Install dependencies
ENV DEBIAN_FRONTEND=noninteractive
RUN apt-get update && apt-get install -y \
    python3 \
    openssh-server

# Switch to Python 3 (3.10 is default in Ubuntu 22.04)
RUN update-alternatives --install /usr/bin/python python /usr/bin/python3 1
RUN update-alternatives --set python /usr/bin/python3

# Create non-root user (no password set -- DOCKSEC-02)
RUN useradd --create-home -s /bin/bash remoteuser

# DOCKSEC-05: Accept ephemeral SSH public key via build argument
ARG SSH_PUBKEY
RUN mkdir -p /home/remoteuser/.ssh && \
    echo "${SSH_PUBKEY}" >> /home/remoteuser/.ssh/authorized_keys && \
    chown -R remoteuser:remoteuser /home/remoteuser/.ssh && \
    chmod 700 /home/remoteuser/.ssh && \
    chmod 600 /home/remoteuser/.ssh/authorized_keys

# Copy over data
ADD --chown=remoteuser:remoteuser src/docker/test/e2e/remote/files /home/remoteuser/files

# Change sshd port to 1234
RUN sed -i '/Port 22/c\Port 1234' /etc/ssh/sshd_config
EXPOSE 1234

# DOCKSEC-02: Disable password authentication
RUN sed -i 's/^#\?PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config && \
    sed -i 's/^#\?KbdInteractiveAuthentication.*/KbdInteractiveAuthentication no/' /etc/ssh/sshd_config && \
    echo "UsePAM no" >> /etc/ssh/sshd_config
RUN passwd -l remoteuser

# DOCKSEC-04: Generate host keys and transfer ownership for non-root sshd
RUN mkdir -p /var/run/sshd && \
    ssh-keygen -A && \
    chown remoteuser:remoteuser /etc/ssh/ssh_host_* && \
    chmod 700 /etc/ssh/ssh_host_* && \
    chown remoteuser:remoteuser /var/run/sshd

# Non-root sshd needs writable PidFile location
RUN echo "PidFile /tmp/sshd.pid" >> /etc/ssh/sshd_config

# DOCKSEC-04: Switch to non-root user (port 1234 is non-privileged -- no setcap needed)
USER remoteuser

CMD ["/usr/sbin/sshd", "-D", "-e"]
```

**File 1: `Makefile` -- Add ephemeral key generation to `run-tests-e2e` target**

In the `run-tests-e2e` target, add SSH key generation BEFORE the remote container `docker buildx build` command (before line 131). Insert after the `echo "Building remote container..."` line:

```makefile
	# DOCKSEC-05: Generate ephemeral SSH key pair for E2E test
	ssh-keygen -t ed25519 -N "" -f /tmp/e2e_test_key -q 2>/dev/null || true
	E2E_SSH_PUBKEY=$$(cat /tmp/e2e_test_key.pub)
```

Then update the remote container build command (lines 131-136) to pass the public key:

Change:
```makefile
	$(DOCKER) buildx build \
		--platform $${SEEDSYNCARR_PLATFORM} \
		--load \
		-t seedsyncarr/test/e2e/remote \
		-f ${SOURCEDIR}/docker/test/e2e/remote/Dockerfile \
		.
```
to:
```makefile
	$(DOCKER) buildx build \
		--platform $${SEEDSYNCARR_PLATFORM} \
		--load \
		-t seedsyncarr/test/e2e/remote \
		--build-arg SSH_PUBKEY="$${E2E_SSH_PUBKEY}" \
		-f ${SOURCEDIR}/docker/test/e2e/remote/Dockerfile \
		.
```

The `-q 2>/dev/null || true` on ssh-keygen suppresses the overwrite prompt if the file already exists from a previous run. The key is ephemeral -- created fresh each `make run-tests-e2e` invocation.

| Boundary | Description |

| myapp container -> remote container | App connects via SSH/lftp to remote container on port 1234 |
| Makefile host -> Docker build context | ssh-keygen generates key on host, pubkey passed as build-arg |
| compose volume -> myapp container | Private key mounted read-only into myapp |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-93-11 | Information Disclosure | Hardcoded password "remotepass" in Docker image | mitigate | Remove chpasswd, disable PasswordAuthentication, lock password (DOCKSEC-02) |
| T-93-12 | Information Disclosure | Static id_rsa.pub committed to git repo | mitigate | Delete file, replace with ephemeral key via --build-arg (DOCKSEC-05) |
| T-93-13 | Elevation of Privilege | sshd running as root in remote container | mitigate | Run as remoteuser, transfer host key ownership (DOCKSEC-04) |
| T-93-14 | Information Disclosure | Ephemeral key at /tmp/e2e_test_key on CI runner | accept | Key is ephemeral per build, CI runner is ephemeral, key has no value after test run |
| T-93-15 | Denial of Service | ssh-keygen fails silently in Makefile | accept | `|| true` prevents overwrite prompt failure; missing key will cause docker build ARG to be empty and SSH auth to fail, caught by E2E test failures |
</threat_model>
