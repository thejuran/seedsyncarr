## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Disk ↔ process | `secrets.key` and any Fernet token pass through `common/encryption.py` between the POSIX filesystem (0600 expected) and the Python process. No network or cross-process IPC touches this module. |
| Process ↔ PyPI | `cryptography` is pulled from PyPI via poetry; the lockfile is the integrity boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-81-01-01 | Information Disclosure | `load_or_create_key` create-path | mitigate | Atomic `os.open(..., os.O_WRONLY \| os.O_CREAT \| os.O_EXCL, 0o600)` — keyfile never exists at looser permissions, even for a single syscall. Ref RESEARCH §9.2. |
| T-81-01-02 | Information Disclosure | `load_or_create_key` existing-file branch | mitigate | `os.chmod(keyfile, 0o600)` on every load (matches persist.py:43 precedent). Tightens permissions even if an operator copied the file with loose mode. |
| T-81-01-03 | Tampering | Fernet ciphertext corruption | mitigate | Fernet's HMAC-SHA256-before-AES-CBC (PyCA recipe layer) detects any bit-flip; `InvalidToken` raised, caught at `decrypt_field` boundary, re-raised as `DecryptionError`. |
| T-81-01-04 | Information Disclosure | `DecryptionError` message | mitigate | Error message is a fixed string ("Failed to decrypt Fernet token") — MUST NOT include the token contents or key material. `from` chain preserves traceback for debugging without leaking to log redactors. |
| T-81-01-05 | Denial of Service | `is_ciphertext` base64 parse on arbitrary input | mitigate | Length gate (>=100) runs BEFORE `base64.urlsafe_b64decode`. Caps the CPU cost of the discriminator on untrusted input; `base64.urlsafe_b64decode` itself is O(n) and non-recursive. |
| T-81-01-06 | Cryptographic | Fernet key strength | transfer | Key generation delegated to `Fernet.generate_key()` (PyCA, `os.urandom` backed). Not our problem to strengthen. |
| T-81-01-07 | Elevation of Privilege | Windows chmod no-op (pitfall 8.3) | accept | Project targets Linux (Debian 12 Docker runtime); CI is `ubuntu-latest`. Windows users get best-effort behavior via `os.chmod`; documented in code comment on `load_or_create_key`. Ref research A5. |
| T-81-01-08 | Supply Chain | `cryptography` dependency | mitigate | Version range `>=44,<47` caps at a tested major boundary; `poetry.lock` pins the exact version + hashes. Dependabot surfaces CVEs in this package per existing workflow. |
