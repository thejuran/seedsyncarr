**PYFIX-18 (busy-wait sleep injection) -- per D-08:**

In `unittests/test_lftp/test_lftp.py`:

1. Add `import time` to the imports block if not already present (currently imports: logging, os, shutil, sys, tempfile, unittest, timeout_decorator, lftp modules -- no `time` import).

2. For ALL 41 `while True:` busy-wait loops in the file, add `time.sleep(0.01)` as the LAST statement in the loop body, AFTER all `if condition: break` checks. This ensures sleep only runs when the loop continues (not when it breaks).

The loops follow consistent patterns:

**Pattern A (most common -- ~35 occurrences):**
```python
# Before:
while True:
    statuses = self.lftp.status()
    if len(statuses) > 0:
        break

# After:
while True:
    statuses = self.lftp.status()
    if len(statuses) > 0:
        break
    time.sleep(0.01)
```

**Pattern B (multi-condition -- ~6 occurrences):**
```python
# Before:
while True:
    statuses = self.lftp.status()
    if len(statuses) > 0 and statuses[0].state == LftpJobStatus.State.RUNNING:
        break

# After:
while True:
    statuses = self.lftp.status()
    if len(statuses) > 0 and statuses[0].state == LftpJobStatus.State.RUNNING:
        break
    time.sleep(0.01)
```

**Pattern C (zero-check -- some occurrences):**
```python
# Before:
while True:
    statuses = self.lftp.status()
    if len(statuses) == 0:
        break

# After:
while True:
    statuses = self.lftp.status()
    if len(statuses) == 0:
        break
    time.sleep(0.01)
```

**Key rules:**
- `time.sleep(0.01)` is ALWAYS the last line before the loop repeats
- Place it AFTER the `if ... break` block, NOT before
- Use exactly `0.01` seconds (10ms) -- sufficient to yield CPU without meaningfully slowing tests
- The file has `@timeout_decorator.timeout(5)` on most test methods, so 41 loops x ~2 iterations x 10ms = ~0.8s is well within budget
- Do NOT add sleep after `break` statements (unreachable code)
- Some loops have multiple `if` checks before the final break -- the sleep goes after ALL break checks
