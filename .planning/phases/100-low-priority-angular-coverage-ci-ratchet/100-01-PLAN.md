<objective>
Add a fakeAsync regression test (COVLOW-03) that pins the SSE heartbeat-vs-timeout race in `StreamDispatchService`: a heartbeat `ping` arriving AFTER the 30s timeout boundary but BEFORE the next 5s checker tick re-arms `_lastEventTime`, so the next `checkConnectionTimeout` must NOT trigger a spurious reconnect and must NOT create a double subscription. Paired with a positive-control test proving the timeout DOES fire (and reconnect) when no heartbeat arrives.

Purpose: The guard under test is that `checkConnectionTimeout` re-reads `_lastEventTime` fresh on every interval (`stream-service.registry.ts:130`), so a heartbeat that updated `_lastEventTime` (line 204) before the next checker tick wins the race. Without this test, a refactor that snapshotted `_lastEventTime` once per interval — or moved the `_lastEventTime = 0` reset earlier — would silently re-introduce spurious reconnects and double subscriptions over already-correct code.

Output: Two new `it()` blocks (a nested `describe("heartbeat-vs-timeout race", ...)`) appended to the existing `describe("Testing stream dispatch service")` in `stream-service.registry.spec.ts`. No new files; no production code change (D-09 — pure regression net, expected green on first run).
</objective>
