4. Properties:
   - `activeSegment: "all" | "active" | "errors" = "all"` — tracks which segmented filter button is active
   - `nameFilter = ""` — tracks search input value

   - `readonly pageSize = 15`

   - Private `pageSubject = new BehaviorSubject<number>(1)`

5. In constructor, set up the paged observable:
   ```typescript
   this.filteredFiles$ = this.viewFileService.filteredFiles;

   const combined$ = combineLatest([this.filteredFiles$, this.pageSubject]);

   this.pagedFiles$ = combined$.pipe(
     map(([files, page]) => {
       const start = (page - 1) * this.pageSize;
       return files.slice(start, start + this.pageSize).toArray();
     })
   );

   this.totalPages$ = this.filteredFiles$.pipe(
     map(files => Math.max(1, Math.ceil(files.size / this.pageSize)))
   );

   this.totalCount$ = this.filteredFiles$.pipe(
     map(files => files.size)
   );
   ```

6. Reset page to 1 when filters change — subscribe to `ViewFileOptionsService.options` and call `this.goToPage(1)` on each emission:
   ```typescript
   this.viewFileOptionsService.options.pipe(takeUntil(this.destroy$)).subscribe(() => {
     this.goToPage(1);
   });
   ```

7. Methods:
   - `onSearchInput(value: string)`: calls `this.viewFileOptionsService.setNameFilter(value)` and sets `this.nameFilter = value`
   - `onSegmentChange(segment: "all" | "active" | "errors")`:
     - Set `this.activeSegment = segment`
     - If `segment === "all"`: call `this.viewFileOptionsService.setSelectedStatusFilter(null)`
     - If `segment === "active"`: call `this.viewFileService.setFilterCriteria(...)` — BUT WAIT: The existing pipeline uses `ViewFileOptionsService.setSelectedStatusFilter()` which only accepts a single status. The "Active" compound filter (DOWNLOADING + QUEUED + EXTRACTING) and "Errors" compound filter (STOPPED + DELETED) need a multi-status filter.
     
     IMPORTANT APPROACH FOR COMPOUND FILTERS: Do NOT bypass ViewFileOptionsService. Instead:
     - For "All": `viewFileOptionsService.setSelectedStatusFilter(null)`
     - For "Active" and "Errors": We need to extend the filter. The cleanest approach within the existing architecture:
       1. Add a new method to `ViewFileOptionsService`: `setCompoundStatusFilter(statuses: ViewFile.Status[] | null)` OR
       2. Simpler: Apply a local filter on top of filteredFiles$ within the TransferTableComponent itself. This avoids modifying the shared service.
     
     USE APPROACH 2 (local filter): Do NOT modify ViewFileOptionsService or ViewFileFilterService. Instead:
     - Keep `viewFileOptionsService.setSelectedStatusFilter(null)` for all segments (clears the service-level status filter)
     - Maintain a local `segmentFilter$: BehaviorSubject<"all" | "active" | "errors">` 
     - Derive `segmentedFiles$` from `combineLatest([this.viewFileService.filteredFiles, this.segmentFilter$])`:
       ```typescript
       this.segmentedFiles$ = combineLatest([
         this.viewFileService.filteredFiles,
         this.segmentFilter$
       ]).pipe(
         map(([files, segment]) => {
           if (segment === "all") return files;
           if (segment === "active") {
             return files.filter(f =>
               f.status === ViewFile.Status.DOWNLOADING ||
               f.status === ViewFile.Status.QUEUED ||
               f.status === ViewFile.Status.EXTRACTING
             ) as Immutable.List<ViewFile>;
           }
           // errors
           return files.filter(f =>
             f.status === ViewFile.Status.STOPPED ||
             f.status === ViewFile.Status.DELETED
           ) as Immutable.List<ViewFile>;
         })
       );
       ```
     - Use `segmentedFiles$` as the source for `pagedFiles$` and `totalPages$` and `totalCount$` instead of `filteredFiles$` directly.
   - `goToPage(page: number)`: sets `this.currentPage = page; this.pageSubject.next(page); this._cdr.markForCheck();`
   - `onPrevPage()`: `if (this.currentPage > 1) this.goToPage(this.currentPage - 1)`
   - `onNextPage(totalPages: number)`: `if (this.currentPage < totalPages) this.goToPage(this.currentPage + 1)`
   - `trackByName(index: number, file: ViewFile): string { return file.name!; }`

8. `ngOnDestroy()`: destroy$.next(), destroy$.complete()
